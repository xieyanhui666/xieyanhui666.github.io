    函数返回值：成功时，返回SQLITE_OK，失败时，返回-1；
    main函数调用它的时候，应该sqlite3 *pDb; sqlite3_open(“aa”, &pDb);
    调用该函数成功返回时，pDb这个指针指向被打开的数据库。

    函数功能：关闭数据库
    函数参数： pDb指向打开的数据库
    函数返回值： 成功时，返回SQLITE_OK，失败时，返回-1；

    函数功能：操作数据库，并且返回结果，一般多用于SQL select语句
    函数参数： pDb指向打开的数据库
    pSql : 将要执行的sql语句
    pResult : 指向返回的结果(查询到的结果，是一个二维数组)
    pRow :指向结果的行数
    pColumn : 指向结果的列数
    pErrmsg : 如果执行失败，将会指向出错消息。如果执行失败，需要调用sqlite3_free进行释放
    函数返回值： 成功时，返回SQLITE_OK，失败时，返回-1；

    函数功能：操作数据库，并且返回结果，一般多用于SQL select语句
    函数参数：
    pResult : 指向sqlite3_get_table返回的结果(查询到的结果)
    函数返回值： 成功时，返回SQLITE_OK，失败时，返回-1；

    函数功能：执行sql语句操作数据库，成功时，调用回调函数。
    函数参数：
    pDb: 指向打开的数据库
     pSql : 将要执行的sql语句

    arg :给回调函数传的第一个参数
    pErrmsg : 如果执行失败，将会指向出错消息。如果执行失败，需要调用sqlite3_free进行释放
    函数返回值： 成功时，返回SQLITE_OK，失败时，返回-1；

    函数功能：释放查询/操作数据库返回的错误信息
    函数参数：pErrmsg : 执行sql语句后的出错信息

​ 5.优先级
​ 6.执行状态和属性

线程间通信

线程共享同一进程的地址空间
优点：线程间通信很容易（通过全局变量交换数据）
缺点：多个线程访问共享时需要同步或互斥机制

同步（synhcronization）

：多个任务按照约定的先后次序相互配合完成一件事情。

互斥

临界资源：一次只允许一个任务（进程、线程）访问的共享资源
临界区：访问临界区的代码
互斥机制：
mutex互斥锁；
任务访问临界资源前申请锁，访问完后释放锁

返回值：成功返回0，失败返回错误码
mutex 指向要初始化的互斥锁对象
attr：互斥锁属性，NULL表示缺醒属性

返回值：成功返回0，失败返回错误码
mutex：指向要初始化的互斥锁对象
如果无法获得锁，任务阻塞

返回值：成功返回0， 失败返回错误码
mutex：指向要初始化的互斥锁对象

当前在访问临界资源以前申请锁，申请到锁之后执行临界区，执行完临界区释放锁（如果没有释放锁，会导致其他申请锁的任务一直阻塞）

多个临界资源
1.临界资源是同时被访问的，只需要一个锁
2. 临界资源不同时候被访问，需要多个互斥锁保护临界资源

（信号量的值为0或者1就可以实现互斥）
注意同步和互斥的区别

当不用互斥的时候，主线程某个时刻会出现将count赋值给value1后，时间片用完了，执行副线程，会打印value1和value2的值。


当启用互斥锁之后，在执行主线程时，每一次循环count的值都会分别赋值给它们，副线程是不能够问value1和value2的值的，反之亦然，所以不会打印两个变量的值。

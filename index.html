编程方法：

  • 寄存器编程：运行效率最高，编程效率低，可移植性最差
  • 标准外设库：稳定，效率比较高，编程效率一般，支持芯片有限，不支持新版本芯片，可移植性较差，已经过时了。
  • HAL/LL库：ST公司主推的函数库，可以使用图形化配置，可移植性较好，性能和稳定性相对较差。

  4. 安装MDK
  5. 安装MDK pack包（STM32F0）
  6. 安装ST-Link驱动

配置CubeMX

orgin：

多任务编程：通俗的讲，用户可以在同一时间运行多个应用程序（实际是并发）。多任务操作系统使用某种调度策略支持多个任务并发执行。
任务：每个正在执行的应用程序，任务是一个逻辑概念，指由软件完成的活动，或者是为实现某个目的一系列操作。通常一个任务是一个程序的一次运行，一个任务包含一个或者多个完成独立功能的子任务，这个独立的子任务是进程或者线程。

why：

    每个应用程序的都可以运行很多次，有多个实例，再用程序去描述程序会显得不合适。
    进程和程序的区别：
程序是静态的代码，是保存在非易失性存储器上的指令和数据的有序集合，没有执行概念。
进程是一个动态的，包括了动态的创建、调度、执行、消亡的整个过程。

how:

    进程是具有相对独立功能的程序在特定数据集合上动态执行的过程。

分类

：
交互式进程：shell命令进程、文本编辑器和图形应用程序运行
批处理进程：编译器的编译操作、数据库搜索引擎
守护进程： （一直在后台运行，不与任何终端的不关联) 系统进程

特点：

并发性:系统中多个进程可以同时并发执行，相互之间不受干扰
动态性：进程都有完整的生命周期，而且在进程的生命周期内，进程的状态是不断变化的，进程具有动态的地址空间（代码、数据、进程控制块）。
交互性：指的是进程间直接和间接的通信
独立性：进程是资源分配和调度的基本单位，各个进程的地址空间是相互独立的。

平台设备框架
设备–硬件（LED 鼠标 键盘 窗口）—》hello.ko
驱动–软件—》driver.ko
使用虚拟总线将设备和它的驱动程序关联起来。虚拟总线称为platform
将挂载在虚拟总线上的设备称为platform_device
将挂载在虚拟总线上的驱动称为platform_driver
当在linux系统中注册一个设备时，会通过虚拟总线去找与之匹配的驱动程序。
当在linux系统中注册一个驱动程序时，会通过虚拟总线去找与之匹配的设备。
初始化流程：—》helloModule
{
1 注册平台设备platform_device_register(struct platform_device * )
(将设备挂载到平台总线上)
2 注册平台驱动platform_driver_register(struct platform_driver *)
（将驱动程序挂载到平台总线上）
}
exit:—》helloExit
{
1 去注册平台驱动platform_driver_unregister(struct platform_driver *)
2 去注册平台设备platform_device_unregister(struct platform_device *)
}
struct platform_device platformDevice;//定义一个平台设备
={ .name=”hello_platform”，平台设备名称
.dev={
.release=HelloPlatformRelease,//当设备被去注册的时候被调用
}
.resource = res,
}
struct resource res[]={
{
.start=0x1c081ffc,
.end=0x1c081fff,
.flags=IORESOURCE_MEM,
}
{
.start=11,
.flags=IORESOURCE_IRQ,
}
}
struct platform_driver platformDriver;//定义一个平台驱动
={
.probe=helloProbe():探测该驱动对应的设备有没有
.remove=helloRemove();移除平台设备
.driver={
.name=”hello_platform”;
.owner=THIS_MODULE;
}
}

互斥

临界资源：一次只允许一个任务（进程、线程）访问的共享资源
临界区：访问临界区的代码
互斥机制：
mutex互斥锁；
任务访问临界资源前申请锁，访问完后释放锁

返回值：成功返回0，失败返回错误码
mutex 指向要初始化的互斥锁对象
attr：互斥锁属性，NULL表示缺醒属性

返回值：成功返回0， 失败返回错误码
mutex：指向要初始化的互斥锁对象

当前在访问临界资源以前申请锁，申请到锁之后执行临界区，执行完临界区释放锁（如果没有释放锁，会导致其他申请锁的任务一直阻塞）

多个临界资源
1.临界资源是同时被访问的，只需要一个锁
2. 临界资源不同时候被访问，需要多个互斥锁保护临界资源

（信号量的值为0或者1就可以实现互斥）
注意同步和互斥的区别

当不用互斥的时候，主线程某个时刻会出现将count赋值给value1后，时间片用完了，执行副线程，会打印value1和value2的值。


当启用互斥锁之后，在执行主线程时，每一次循环count的值都会分别赋值给它们，副线程是不能够问value1和value2的值的，反之亦然，所以不会打印两个变量的值。

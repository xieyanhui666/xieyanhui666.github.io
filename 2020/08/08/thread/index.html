一个进程中的多个线程共享以下资源：
​ 1.可执行的指令
​ 2.静态数据
​ 3.进程中打开的文件描述符
​ 4.当前工作目录
​ 5.用户ID
​ 6.用户组ID
每个线程私有的资源：
​ 1.线程ID
​ 2.PC和相关寄存器
​ 3.堆栈
​ 4.错误号（errno）
​ 5.优先级
​ 6.执行状态和属性

线程间通信

线程共享同一进程的地址空间
优点：线程间通信很容易（通过全局变量交换数据）
缺点：多个线程访问共享时需要同步或互斥机制

同步（synhcronization）

：多个任务按照约定的先后次序相互配合完成一件事情。
信号量：代表某一类资源，其值表示系统中该资源的数量
分类：无名信号量（基于内存的信号量）
有名信号量
操作：
初始化：

返回值：成功返回0，失败EOF
sem：指向要初始化的信号量对象
pshared 0-线程间 1-进程间
val：信号量的初值
P操作（申请资源）

返回值：成功返回0，失败EOF
V操作（释放资源）

返回值：成功返回0，失败EOF

P(S):
if （信号量的值大于0）
{
申请资源的任务继续运行；
信号量的值减一；
}
else
{
申请资源的任务阻塞；
}
V(S):
信号量的值加一；
if（有任务在等待资源）
{
唤醒等待任务，让其继续运行；
}

互斥

临界资源：一次只允许一个任务（进程、线程）访问的共享资源
临界区：访问临界区的代码
互斥机制：
mutex互斥锁；
任务访问临界资源前申请锁，访问完后释放锁

返回值：成功返回0，失败返回错误码
mutex 指向要初始化的互斥锁对象
attr：互斥锁属性，NULL表示缺醒属性

返回值：成功返回0，失败返回错误码
mutex：指向要初始化的互斥锁对象
如果无法获得锁，任务阻塞

返回值：成功返回0， 失败返回错误码
mutex：指向要初始化的互斥锁对象

当前在访问临界资源以前申请锁，申请到锁之后执行临界区，执行完临界区释放锁（如果没有释放锁，会导致其他申请锁的任务一直阻塞）

多个临界资源
1.临界资源是同时被访问的，只需要一个锁
2. 临界资源不同时候被访问，需要多个互斥锁保护临界资源

（信号量的值为0或者1就可以实现互斥）
注意同步和互斥的区别

当不用互斥的时候，主线程某个时刻会出现将count赋值给value1后，时间片用完了，执行副线程，会打印value1和value2的值。


当启用互斥锁之后，在执行主线程时，每一次循环count的值都会分别赋值给它们，副线程是不能够问value1和value2的值的，反之亦然，所以不会打印两个变量的值。
